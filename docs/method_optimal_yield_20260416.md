### 2.4 区域最优产量识别与情景评价

在完成产量预测模型筛选后，本文进一步基于最优模型识别不同区域在当前环境背景下的最优管理方案及其对应的最大潜在产量。该过程采用“区域级反事实模拟 + 全局优化”的思路，即保持各地块的环境条件和其他非管理背景变量不变，仅对可调控管理变量进行搜索，并将训练好的产量模型作为区域产量响应函数。设区域 \(r\) 内第 \(i\) 个地块在候选管理方案 \(x\) 下的预测产量为

\[
\hat{Y}_{ir}(x)=f(X_{ir}(x))
\]

其中，\(f(\cdot)\) 表示训练后的最优产量预测模型，\(X_{ir}(x)\) 表示在保留地块原始环境背景的前提下，用候选管理值替换相关管理变量后形成的输入特征向量。为了获得区域层面的代表性最优方案，本文将区域目标定义为区域内全部地块预测产量的中位数，并加入经验分布约束项：

\[
J_r(x)=\mathrm{median}_i\{\hat{Y}_{ir}(x)\}-\lambda D_r(x)
\]

其中，\(D_r(x)\) 为候选方案 \(x\) 与该区域历史管理样本之间的标准化最小距离，\(\lambda\) 为惩罚系数。引入该惩罚项的目的是限制优化结果偏离观测数据支撑范围过远，降低模型在样本稀疏区域外推时产生不合理高值的风险。使 \(J_r(x)\) 最大的管理方案记为 \(x_r^*\)，其对应的区域最优产量定义为

\[
Y_r^*=\mathrm{median}_i\{\hat{Y}_{ir}(x_r^*)\}
\]

本研究直接参与优化的管理变量包括播期（`Sow_DOY`）、种植密度（`Density`）、施氮量（`Fer_N`）、施磷量（`Fer_P`）、施钾量（`Fer_K`）、植保投入（`Pest_Cost`）、灌溉投入（`Irr_Elec`）以及灌溉方式。由于灌溉方式属于离散变量，优化时首先分别固定为无灌溉、漫灌、喷灌和滴灌四种情景，然后在每一种灌溉方式下对其余数值型管理变量进行连续搜索，最终比较不同灌溉方式下的最优结果，并以预测产量最高者作为该区域的最终最优管理方案。

为避免重复优化和变量间逻辑冲突，施肥次数（`Fer_Count`）、打药次数（`Pest_Count`）和灌溉次数（`Irr_Count`）不作为独立优化变量，而是由对应投入强度反推得到。具体而言，`Fer_Count` 根据总养分投入强度 `Fer_N + Fer_P + Fer_K` 反推，`Pest_Count` 根据植保投入 `Pest_Cost` 反推，`Irr_Count` 根据对应灌溉方式下的灌溉投入 `Irr_Elec` 反推。反推过程采用单调回归（isotonic regression）建立“投入强度-作业次数”关系，以保证随着投入增加，反推得到的作业次数不会出现不合理下降，从而提高管理解释的一致性和情景可执行性。

由于区域目标函数依赖于机器学习模型预测结果，具有明显的非线性、非光滑和不可导特征，因此本文采用差分进化算法（differential evolution）进行全局优化。该算法在给定边界内迭代搜索候选解，适合处理黑箱目标函数和多峰优化问题。为保证优化结果具有现实可执行性，所有直接优化变量在每次评估前均进行量化处理：播期按 1 d 取整，密度、养分投入、植保投入和灌溉投入按照预设管理步长量化后再送入产量模型和次数反推模块。与此同时，各变量的搜索范围限定在区域历史观测分布的经验分位区间内，从而避免超出样本支撑范围的极端外推。无灌溉情景下进一步强制设定 `Irr_Elec = 0` 且 `Irr_Count = 0`。

在情景评价阶段，本文以区域观测样本的中位数作为当前管理基线，分别计算基线情景和最优情景下的区域产量、成本和利润。成本数据统一整理为公顷尺度，仅保留播种成本、灌溉设备成本、灌溉运行成本、施肥成本和总成本等必要字段。随后分别建立播种成本、施肥成本和灌溉运行成本的线性子模型，并结合不同灌溉方式对应的设备成本中位数以及固定成本项，估计任一候选管理方案下的总投入成本。在此基础上，结合预测产量进一步计算利润水平。最终，通过比较区域基线情景与最优情景的产量、成本和利润差异，量化在现有环境背景下仅通过管理调整所能实现的潜在增产空间。

需要说明的是，本文得到的最优产量和最优管理方案属于模型驱动的情景估计结果，而非田间干预试验直接验证的因果效应。因此，该方法更适用于区域尺度管理优化潜力的识别与关键调整方向的比较，而不应直接等同于单地块层面的精确处方推荐。
