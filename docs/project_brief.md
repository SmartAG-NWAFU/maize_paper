# 项目摘要

## 论文主线

这篇论文的核心不是单纯解释“什么因素影响产量”，而是量化“在当前管理基础上，还能提升多少产量和利润”。研究对象为黄淮海地区玉米生产样本，分析框架为：

1. 构建泄漏可控的玉米产量预测模型。
2. 以当前管理为基线定义参照情景。
3. 仅在观测数据支持的范围内调整可控管理变量。
4. 搜索最高产量方案并量化增产幅度。
5. 根据情景成本同步核算利润变化。

## 关键结论草案

- 整体单产由 `794.70 kg/亩` 提高至 `914.56 kg/亩`，增幅 `15.08%`。
- 整体利润由 `1236.18 元/亩` 提高至 `1516.70 元/亩`，增幅 `22.69%`。
- 河北单产和利润增幅高于山东，说明当前管理基础较低区域具备更大优化潜力。
- 最高产量方案的核心调整方向是播期前移、提高种植密度、优化氮钾投入和强化灌溉保障。

## 当前数据对象

`data/model_vars.csv` 当前为 81 个地块级观测，包含以下字段：

- 标识变量：`uuid`
- 区域变量：`Region`
- 管理变量：`Sow_DOY`、`Density`、`Fer_N`、`Fer_P`、`Fer_K`、`Fer_Count`、`Pest_Count`、`Pest_Cost`、`Irr_Count`、`Irr_Elec`
- 灌溉方式变量：`Irr_Sprinkler`、`Irr_None`、`Irr_Drip`、`Irr_Flood`
- 田间状态变量：`Lodging`（倒伏率）
- 天气变量：`Precip`、`Rad`、`RH`、`Wind`（生长季最大风速）
- 目标变量：`Yield`

变量统计摘要已同步写入 `docs/model_variable_table.md` 和 `docs/model_variable_table.csv`。

`data/cost.csv` 当前可支撑：

- 租金、播种、灌溉、施肥、打药成本提取
- 区域标签对照
- 总成本核算

## 初始化后的项目约定

- 统一使用 `outputs/` 存放模型结果和分析产物。
- `src/model.py` 作为建模主入口，默认目标列为 `Yield`，并兼容 `yield_ha`、`yield_per_mu` 作为候选名称。
- 利润在后续脚本中基于产量情景和成本数据单独核算，避免目标泄漏和解释争议。
- 论文分析组织方式遵循 “yield-first, profit-second”。

## 当前关键脚本

1. `src/model.py`: 训练产量预测模型并输出评估结果与特征重要性。
2. `src/optimize_management_yield.py`: 定义基线管理并搜索最高产量方案，同时核算利润变化。
3. `src/plot_management_responses.py`: 输出主要管理变量的响应曲线。
4. `src/figs/*.py`: 生成论文图件，包括播期、施肥、灌溉、打药、收益和特征重要性图。
